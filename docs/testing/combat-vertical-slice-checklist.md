# 전투 버티컬 슬라이스 검증 체크리스트

검증일: 2026-06-22

검증 환경: Node.js 22.14.0, npm 10.9.2, Vite 8.0.16, 인앱 브라우저

## 전투 규칙과 결과

- [x] 초기 상태: 라운드 1, 아군과 적군의 최대 체력, 선택되지 않은 영웅을 브라우저로 확인했고 빈 전투 이벤트 상태를 자동 테스트로 확인했다.
- [x] 적 행동 예고: 숲쥐 A, 숲쥐 B, 이끼 슬라임의 초기 의도 3개와 피해량을 확인했다.
- [x] 라운드당 1회 행동: 행동을 마친 영웅의 선택 버튼이 비활성화되고 다음 라운드에서 다시 활성화되는 것을 확인했다.
- [x] 고정 피해: 사격 피해 3으로 숲쥐 A가 HP 3/3에서 0/3이 되는 것을 확인했고 자동 테스트로 스킬 피해 계산을 확인했다.
- [x] 쓰러진 대상: HP 0인 적의 공격 대상 버튼이 비활성화되는 것을 확인했다.
- [x] 쓰러진 적 행동 제외: 숲쥐 A를 쓰러뜨린 뒤 턴을 종료했을 때 궁수 HP가 8/8로 유지되고 생존한 두 적만 행동하는 것을 확인했다.
- [x] 턴 종료 버튼: 영웅 행동 도중과 모든 행동 완료 뒤 사용할 수 있고, 적 행동을 해결한 뒤 다음 라운드로 전환되는 것을 확인했다.
- [x] 승리 결과: 결정론적 브라우저 진행으로 승리 대화상자, 재시작 버튼 초점, Escape 취소 방지를 확인했고 자동 테스트도 통과했다.
- [x] 패배 결과: 적 행동 전·후의 패배 판정과 네이티브 패배 대화상자 경로를 기존 자동 테스트로 확인했다.
- [x] 재시작: 라운드 1, 모든 체력, 초기 의도, 선택 해제를 브라우저로 확인했고 빈 이벤트와 컨트롤러 선택 초기화를 자동 테스트로 확인했다.

## 화면과 접근성

- [x] 데스크톱 1280x720: 제목, URL, 스테이지 제목, 의도, HP, 행동 선택, 승리, 재시작 전체 흐름을 확인했다.
- [x] 모바일 가로 844x390: 가로·세로 페이지 오버플로가 없고 HP, 의도, 행동 버튼이 읽히며 모든 버튼이 최소 44px이고 행동 1회와 턴 종료를 완료했다.
- [x] 모바일 가로 667x375: 가로·세로 페이지 오버플로가 없고 HP, 의도, 행동 버튼이 읽히며 모든 버튼이 최소 44px이고 행동 1회와 턴 종료를 완료했다.
- [x] 모바일 세로 390x844: 네이티브 방향 안내 대화상자에 초점이 있고 게임 영역이 `inert`와 `aria-hidden="true"` 상태임을 확인했다. 가로로 돌아온 뒤 라운드와 체력이 유지되고 턴 종료 버튼으로 초점이 복원됐다.
- [x] 이모지 없음: 브라우저의 표시 텍스트 검사와 기존 자동 테스트에서 이모지 문자가 없음을 확인했다.

## 브라우저와 에셋 상태

- [x] 콘솔 상태: 전체 브라우저 흐름에서 오류와 경고가 없었다.
- [x] 캐릭터 이미지: 표시된 이미지 6개가 모두 로드 완료 상태이며 `naturalWidth`와 `naturalHeight`가 각각 256이었다.
- [x] 배경 이미지: `goblin-forest.png`가 계산된 배경 스타일과 로드된 페이지 에셋 목록에 존재했다.
- [x] 픽셀 아트: 영웅과 적 모두 실제 픽셀 아트로 표시되어 플레이스홀더가 남아 있지 않았다.
