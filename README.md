# 시뮬레이션게임

세 캐릭터를 직접 지휘하고 증강 시너지를 쌓아 5개 스테이지를 돌파하는 턴제 로그라이크 웹게임입니다.

현재 React + TypeScript 프론트엔드에 1-1 전투 버티컬 슬라이스가 구현되어 있습니다. 적 행동 예고, 세 영웅의 행동 선택, 생존한 적의 반격, 승리·패배 결과, 재시작, 모바일 가로 모드와 세로 모드 안내를 포함합니다. 승인된 상세 설계는 [프로토타입 설계 문서](docs/superpowers/specs/2026-06-20-turn-based-roguelike-design.md)에 정리되어 있으며, 현재 검증 상태는 [전투 버티컬 슬라이스 체크리스트](docs/testing/combat-vertical-slice-checklist.md)에서 확인할 수 있습니다.

## 핵심 방향

- 전사, 궁수, 법사로 이루어진 3인 파티
- 3개 공격로와 전·후방 위치를 사용하는 턴제 전투
- 적의 다음 행동 확인 후 아군 세 명을 원하는 순서로 직접 조작
- 고정 스킬에 증강을 결합해 작동 방식과 피해 배수를 변화
- 빌드 폭발의 재미 60%, 적 행동을 읽는 전술 퍼즐 40%
- 피해량은 작고 고정된 수로 시작하고, 희귀한 시너지에만 곱연산 사용
- 죽으면 런 능력치는 초기화되지만 새 콘텐츠와 정보는 영구 해금
- 첫 프로토타입은 챕터 1의 5개 스테이지와 10~15분 분량

## 챕터 1

배경은 `고블린 숲길`이며 1-5 보스는 `고블린 전차대장`입니다.

1. 1-1: 기본 조작과 적 행동 예고
2. 1-2: 전·후방, 밀치기, 관통
3. 1-3: 표식, 광역 공격, 행동 지연
4. 1-4: 정예 전투와 빌드 점검
5. 1-5: 두 단계로 진행되는 보스전

1-2부터 1-4까지는 안전한 조건과 위험한 조건 중 하나를 선택합니다. 위험한 조건으로 승리하면 희귀·전설 증강의 등장 가능성이 높아집니다.

## 확정된 전투 규칙

- 라운드 순서: 적 행동 예고 → 아군 세 명 자유 순서 행동 → 생존한 적 전체 행동
- 일반 스킬: 개별 쿨타임
- 필살기: 팀 공용 게이지를 한 캐릭터가 소비
- 피해는 고정이며 기본 치명타·회피·피해 범위 랜덤 없음
- 생존 캐릭터는 스테이지 종료 시 최대 체력의 25% 회복
- 세 캐릭터가 모두 쓰러지면 런 패배
- 쓰러진 캐릭터는 다음 증강 보상을 포기하면 체력 25%로 부활
- 같은 증강을 다시 선택하면 2단계로 강화
- 캐릭터당 전투 중 1회 무료 위치 변경은 플레이테스트 후 확정

## 기술 방향

- 게임 및 관리자 화면: React + TypeScript
- API: Spring Boot
- 데이터베이스: PostgreSQL
- 스키마 변경: Flyway
- 로컬 환경: Docker Compose 지원
- 형상 관리: Git

Windows 데스크톱과 macOS 노트북에서 같은 저장소를 사용할 수 있게 운영체제별 절대 경로를 피합니다. Docker는 필수가 아니라 편한 실행 수단입니다. 개발 중에는 React와 Spring Boot를 로컬에서 실행하고 PostgreSQL만 Docker로 실행할 수도 있습니다.

PC와 모바일을 처음부터 함께 검증합니다. 모바일 전투 화면은 가로 모드를 기준으로 합니다.

## 로컬에서 첫 전투 실행

Node.js 22.12 이상과 npm 10 이상이 필요합니다.

```powershell
cd frontend
npm install
npm run dev -- --host 0.0.0.0
```

Vite가 출력하는 `Local` URL로 데스크톱에서 접속할 수 있습니다. 같은 Wi-Fi에 연결된 모바일 기기에서는 `Network` URL로 접속하고 가로 모드로 플레이합니다.

전체 자동 검증은 프론트엔드 디렉터리에서 실행합니다.

```powershell
npm run check
```

현재 버티컬 슬라이스는 프론트엔드만으로 실행되므로 Docker, Spring Boot, PostgreSQL은 아직 필요하지 않습니다.

## 플레이 로그와 밸런스 분석

개인정보 없이 브라우저에 저장된 익명 기기 ID로 런을 연결합니다. 다음 원본 이벤트를 저장합니다.

- 안전·위험 경로의 제시와 선택
- 증강 3개의 제시, 선택, 중복 강화
- 매 턴 캐릭터, 스킬, 대상, 피해, 상태 변화
- 캐릭터 사망과 부활
- 스테이지 결과와 런 최종 결과
- 게임 데이터 및 빌드 버전

분석 화면에서는 비율만 보여주지 않고 원본 건수와 분모를 함께 표시합니다.

예: `제시 127회 / 선택 31회 / 선택 후 보스 도달 18회 / 클리어 7회`

선택률과 성공률은 원본 건수에서 계산합니다. 비공개 밸런스 연구실에서는 진행 퍼널, 증강 단독·조합 성과, 위험 보상 성과, 특정 런 재생, CSV 내보내기와 제한된 읽기 전용 분석 질의를 제공합니다.

## 개발 원칙

- 재미있는 1-1 전투를 먼저 만든 뒤 1-5까지 확장
- 게임 규칙과 UI, 로그 수집, 분석 기능을 분리
- 전투 계산과 증강 효과를 자동 테스트
- PC와 모바일 가로 화면에서 전체 런을 브라우저로 검증
- 한 작업이나 검증이 오래 막히면 더 작은 단위로 분리
- 실제 게임에는 이모지를 사용하지 않고 통일된 AI 픽셀 아트를 사용
